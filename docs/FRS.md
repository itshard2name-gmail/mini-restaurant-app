# 功能需求規格書 (FRS) - 餐廳營運管理系統

**版本**: 1.0  
**日期**: 2026-01-03  
**作者**: 產品經理 (PM)  
**目標讀者**: 餐廳業主、店長、營運管理人員

---

## 1. 文件目的
本文件旨在詳列「Mini Restaurant App」的系統功能，並以**現場營運流程**為核心，協助餐廳業主確認系統是否能滿足日常開店、點餐、廚房製作到結帳的實際需求。

## 2. 系統概述
本系統是一套整合「顧客自助點餐」與「後台營運管理」的雲端餐飲系統。設計重點在於**減少外場人力需求**、**優化內場出餐順序**，並提供即時的**數據分析**協助決策。

---

## 3. 功能模組詳解

### 3.1 顧客點餐系統 (Front of House)
*目標：讓顧客無需教學即可直覺完成點餐，減少服務生桌邊點餐時間。*

| 功能項目 | 功能描述 | 營運價值 |
| :--- | :--- | :--- |
| **3.1.1 雙模式首頁** | 系統啟動時讓顧客選擇「內用 (Dine-in)」或「外帶 (Takeout)」。 | 分流不同需求的顧客，內用自動帶入桌號邏輯。 |
| **3.1.2 智慧菜單瀏覽** | • **分類過濾**：點擊「主餐」、「前菜」快速切換。<br>• **即時搜尋**：輸入關鍵字（如 "牛"）即時篩選菜色。<br>• **圖片展示**：大圖示呈現菜品，提升點餐慾望。 | 提升點餐效率，減少顧客猶豫時間，提高客單價。 |
| **3.1.3 購物車與備註** | • 顧客可增減數量。<br>• **口味備註**：每道菜可輸入客製化需求（如：不要蔥、微辣）。 | 減少因「客製化需求」遺漏導致的客訴與重做成本。 |
| **3.1.4 快速結帳流程** | • **外帶登入**：支援手機號碼「快速登入」，自動註冊會員。<br>• **內用桌號**：內用模式下，訂單自動綁定桌號，送餐不送錯。 | 建立會員資料庫（外帶），確保內用送餐準確性。 |
| **3.1.5 訂單狀態追蹤** | 顧客可在「我的訂單」查看進度（待確認 -> 製作中 -> 可取餐）。 | 減少顧客因等候焦慮而頻繁詢問進度的狀況。 |
| **3.1.6 顧客自行取消** | 在店家尚未「接單」前，顧客可自行取消錯誤訂單。 | 節省外場人員處理錯誤訂單的時間。 |

### 3.2 門市營運後台 (Back of House)
*目標：打造「營運指揮中心 (Command Center)」，透過一個畫面掌控全店狀態。*

#### 3.2.1 訂單管理看板 (Order Management)
這是店長與櫃台人員最常使用的畫面，採用類似 Trello 的看板設計，分為五種狀態：

1.  **待處理 (Pending)**：
    *   新訂單進來時會有即時通知（Websocket）。
    *   櫃台人員確認內容無誤後，點擊「接單 (Accept)」。
2.  **製作中 (Kitchen)**：
    *   已接單的項目，廚房開始製作。
    *   狀態對應：系統同時通知顧客「製作中」。
3.  **可取餐 (Counter)**：
    *   製作完成後，點擊「完成 (Ready)」。
    *   狀態對應：外帶顧客收到「可取餐」通知；內用服務生收到「送餐」指示。
4.  **歷史紀錄 (History)**：
    *   查詢已完成或已取消的訂單。
    *   支援依「訂單編號」、「會員電話」或「日期」搜尋。

#### 3.2.2 菜單與庫存管理 (Menu Management)
*   **即時上下架**：售完商品可立即下架，避免持續被點單。
*   **新增菜色**：店長可自行上傳圖片、設定價格與說明。
*   **AI 輔助**：內建 "Magic Fill" 功能，可自動產生菜色描述圖（示意），加快上架速度。

#### 3.2.3 營運數據分析 (Analytics Dashboard)
*提供即時戰情總覽，而非僅是月報表。*

*   **營收總覽**：今日總營業額、總訂單數。
*   **熱門排行**：Top 5 銷售品項，協助備料決策。
*   **通路分析**：圓餅圖分析「內用 vs 外帶」比例，了解客群結構。
*   **時段趨勢**：折線圖顯示各時段營收，協助排班人力調度。

## 4. 系統非功能需求 (NFR)
*除了功能以外，我們承諾的系統品質。*

1.  **全球時區支援**：系統底層採用 UTC 標準時間，若您未來展店至不同國家（或分店跨時區），營收報表依然準確無誤。
2.  **資料安全性**：
    *   顧客密碼採用銀行級加密 (RSA + BCrypt)。
    *   後台權限管控 (RBAC)，工讀生無法隨意更改管理設定。
3.  **高可用性**：即使其中一個微服務（如通知服務）短暫異常，核心點餐功能仍可運作。

## 5. 待確認事項 (Q&A)
*以下項目需要與您進一步討論確認：*

1.  **出單機整合**：目前系統以「螢幕顯示」為主。貴店廚房是否習慣看螢幕 (KDS)，還是必須搭配熱感應出單機？
2.  **支付串接**：目前為「結帳 (Checkout)」流程，是否需直接串接金流（如 LinePay, 信用卡）？目前版本預設為櫃台結帳。
3.  **桌位管理**：目前支援「輸入桌號」，是否需要「視覺化桌圖」（點擊桌子圖示點餐）？
